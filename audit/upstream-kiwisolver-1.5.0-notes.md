## Packaging Notes
- Build backend: `setuptools.build_meta` with `setuptools_scm` and `cppy`.
- Native module import path: `kiwisolver._cext`.
- Python package root: `py/kiwisolver/`.
- Vendorable unchanged: `py/kiwisolver/exceptions.py`, `py/kiwisolver/py.typed`, and the API stub `py/kiwisolver/_cext.pyi` as a reference contract.
- Likely needs local edits: `py/kiwisolver/__init__.py`, packaging files, and the entire `py/src/` native extension layer.
- Compatibility corpus note: the audited release snapshot contains a C++ test corpus only; there is no standalone Python upstream test suite in the inspected release tree, so the port will need adapted C++ solver tests plus new Python compatibility checks derived from the audited API surface.
- Audit provenance: direct `pip download` from PyPI failed in the sandbox with DNS resolution errors, so this audit was reconstructed from the upstream `nucleic/kiwi` release snapshot at commit `5e76d91fd77dc443cb0db36e7398fc13844a0524`, the PyPI release metadata, and local CPython inspection of the native surface shape where available. It is not yet release-verified against the exact PyPI sdist bytes.

## Audit Outcome
- Standalone port remains viable: yes
- Reason: the native surface is compact and centered on five Python-visible solver types plus a small exception set, and the tests exercise solver semantics rather than a broad framework API.
- Candidate Rust solver crates: Cassowary-compatible solver crate with PyO3 binding layer.
- Native API breadth summary: `Variable`, `Term`, `Expression`, `Constraint`, `Solver`, `strength`, version attributes, and six exception classes.
- Test suite size summary: 7 C++ test files under `tests/`, focused on core solver behavior, strengths, edit variables, constraints, and dumps.
- Compatibility plan: adapt the upstream C++ tests into the Rust/host test harness and add Python smoke/compatibility tests from the audited API surface; do not assume an upstream Python corpus exists until one is separately located.
